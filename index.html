Problem Statements
01
Privacy-Preserving Contract Intelligence Engine
NER + Structured Extraction LegalTech / Enterprise
Problem

Law firms and corporate legal teams cannot send confidential contracts to cloud LLMs due to GDPR, attorney-client privilege, and data residency laws. Critical metadata — party names, liability clauses, jurisdiction, payment milestones — remains locked in unstructured PDFs.

Challenge

Use Smolify to synthesize a contract dataset and train a compact NER + clause-extraction model that runs on-device. Output structured JSON: parties, dates, monetary values, obligations, governing law — with zero bytes leaving the user's machine.

Evaluation Criteria
  • F1 score on held-out contract extraction benchmark
  • Inference under 500ms on a mid-range laptop CPU
  • Zero external API calls during inference
Bonus
⚡Support multi-language contracts (Hindi + English code-mixed)
02
Intelligent Clinical Symptom Router
Classification HealthTech / Hospital Ops
Problem

OPDs in tier-2 Indian hospitals receive thousands of patient intake forms daily. Triage staff manually route patients to departments, causing bottlenecks. Sending raw patient data to cloud APIs violates India's DPDP Act.

Challenge

Build a Smolify-trained classification model mapping free-text symptom descriptions to medical departments. Handle regional code-mixing (Hinglish), medical abbreviations, and multi-label outputs. e.g. "chest tightness + cold sweat" → Cardiology

Evaluation Criteria
  • Top-1 and Top-3 accuracy on multi-department test set
  • Graceful ambiguous input handling with confidence score output
Bonus
⚡Integrate urgency scoring: Emergency / Urgent / Routine
03
Financial Document Anomaly Detector
Classification + Reasoning FinTech / Audit / Compliance
Problem

CA firms process thousands of invoices and purchase orders. Fraudulent documents contain subtle inconsistencies: GST mismatches, inflated unit prices, round-number bias, duplicate invoice IDs. Cloud AI cannot access sensitive financial data.

Challenge

Build a Smolify-trained model ingesting structured financial fields (OCR-extracted) and outputting: Clean / Suspicious / Fraudulent with a reasoning trace. Model must learn Benford's Law patterns, vendor fuzzy duplicates, and date sequencing violations.

Evaluation Criteria
  • Precision and Recall on a balanced fraud detection benchmark
  • Quality of reasoning trace (human-rated rubric)
  • False positive rate below 5% on clean document set
Bonus
⚡Generate an audit-ready one-line explanation per flagged field
04
Real-Time Meeting Action Item Extractor
Extraction + Summarization Productivity / Enterprise
Problem

Enterprise meetings generate transcripts that are never acted upon. Action items and owners are buried in hours of text. Sending strategic meeting transcripts to cloud APIs violates corporate data policies. Existing summarizers produce prose blobs, not task lists.

Challenge

Build a Smolify-trained model processing transcript chunks and outputting: [{ action, owner, deadline, priority, context }]. Handle indirect ownership ("let's have the DevOps team handle this") and implicit deadlines ("before next sprint").

Evaluation Criteria
  • Action item recall and precision vs. human-annotated benchmark
  • Owner attribution accuracy
  • Implicit vs. explicit deadline normalization accuracy
Bonus
⚡Classify action items by function: Engineering / Sales / Design / Management
05
Open Innovation Track — Build Your Own Sovereign AI Solution
Open Track Any Domain Your Choice
Vision

The best ideas often come from the ground up. If you've identified a problem space where on-device AI can deliver real value — but it doesn't fit into our predefined tracks — this is your chance to prove it.

Your Challenge

Define your own problem statement, build a Smolify-trained SLM that solves it, and demonstrate measurable impact. The core requirement remains unchanged: own the weights, not the API key. Your model must run entirely on-device or on-premise with zero dependency on cloud inference APIs.

Evaluation Criteria
  • Problem relevance: Does it solve a real pain point for a defined user base?
  • Technical execution: Model quality, inference speed, and deployment feasibility
  • Innovation: Novelty of approach, domain, or integration method
  • Completeness: Working prototype with clear documentation and demo
Requirements

Your submission must include: (1) A clear problem statement explaining the real-world use case, (2) A Smolify-trained model with documented training process, (3) A working demo/prototype showing the model in action.
